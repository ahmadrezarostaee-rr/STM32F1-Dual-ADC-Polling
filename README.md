# STM32F1-Dual-ADC-Polling
Bare-metal STM32F1 firmware that reads two ADC channels (PA0 &amp; PA1) in polling mode using STM32 HAL
STM32F1 Dual ADC Polling

فریمور ساده برای میکروکنترلرهای سری STM32F1 که با استفاده از کتابخانه‌ی STM32 HAL دو کانال ADC را به‌صورت متوالی و در حالت Polling می‌خواند. این پروژه با STM32CubeIDE ساخته شده و می‌تواند به‌عنوان نقطه‌ی شروع برای پروژه‌هایی که به خوانش چند سنسور آنالوگ نیاز دارند استفاده شود.

✨ ویژگی‌ها


خوانش دو کانال ADC (ADC_CHANNEL_0 و ADC_CHANNEL_1) به‌صورت جداگانه و توالی
استفاده از حالت Polling برای سادگی در پیاده‌سازی (بدون نیاز به Interrupt یا DMA)
پیکربندی کلاک سیستم بر پایه‌ی کریستال خارجی (HSE) با PLL
تنظیم یک پایه‌ی GPIO خروجی (PB15) به‌عنوان نمونه
کد تولید شده توسط STM32CubeIDE و قابل گسترش در بخش‌های USER CODE

⚙️ نحوه‌ی کار برنامه

برنامه پس از مقداردهی اولیه‌ی پایه‌ها، ساعت سیستم و واحد ADC، در یک حلقه‌ی بی‌نهایت دو تابع زیر را به‌ترتیب صدا می‌زند:


ADC_Select_CH0() — کانال ۰ را پیکربندی کرده، یک تبدیل را شروع می‌کند، منتظر اتمام آن می‌ماند و مقدار خوانده‌شده را در متغیر سراسری ADC_VAL1 ذخیره می‌کند.
ADC_Select_CH1() — همین فرآیند را برای کانال ۱ تکرار کرده و نتیجه را در ADC_VAL2 ذخیره می‌کند.


این دو متغیر (ADC_VAL1 و ADC_VAL2) می‌توانند در ادامه برای نمایش روی LCD، ارسال سریال، یا هر پردازش دیگری استفاده شوند (در حال حاضر فقط خواندن مقدار انجام می‌شود).

STM32F1 Dual ADC Polling

A minimal firmware project for STM32F1 series microcontrollers that reads two ADC channels sequentially in polling mode using the STM32 HAL library. Generated with STM32CubeIDE, this project can serve as a starting point for applications that need to read multiple analog sensors.

✨ Features


Reads two ADC channels (ADC_CHANNEL_0 and ADC_CHANNEL_1) one after another
Uses polling mode for simplicity (no interrupts or DMA required)
System clock configured from an external HSE crystal with PLL
One GPIO output pin (PB15) configured as a sample/status pin
Code generated by STM32CubeIDE, easily extendable inside the USER CODE sections

⚙️ How It Works

After initializing the GPIOs, the system clock, and the ADC peripheral, the program enters an infinite loop that calls two functions in sequence:


ADC_Select_CH0() — configures channel 0, starts a conversion, waits for it to finish, and stores the result in the global variable ADC_VAL1.
ADC_Select_CH1() — repeats the same process for channel 1 and stores the result in ADC_VAL2.


These two variables (ADC_VAL1 and ADC_VAL2) can later be used for display on an LCD, transmission over UART, or any other processing (currently the firmware only performs the reads).

🚀 Build & Flash


Open the project in STM32CubeIDE (or a similar IDE).
Make sure the HAL drivers for the STM32F1 family are installed.
Build the project and flash it to your STM32F1 board (e.g. using ST-Link).
Inspect ADC_VAL1 and ADC_VAL2 using the debugger, or extend the code to send them over UART.


🛣️ Ideas for Future Development
